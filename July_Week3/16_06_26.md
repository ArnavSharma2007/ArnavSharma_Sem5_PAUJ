## Collection Framework
### List
- Array List: Dynamic Size, Fast Random Access, Duplicate elements allowed
```java
import java.util.ArrayList;

public class Main {
    public static void main(String args[]) {
        ArrayList<String> students = new ArrayList<>();
        students.add("Arnav");
        students.add("Adatyia");
        students.add("Ridhimma");
        students.add("Kanan");
        System.out.println("Original list: ");
        System.out.println(students);

        students.add(2, "Aditya");
        System.out.println(students);

        students.set(1, "Bhavik");
        System.out.println(students);

        students.remove("Arnav");
        System.out.println(students);

        System.out.println("Contains Arnav: " + students.contains("Arnav"));

        System.out.println("---- Final list ----");
        for(String name: students){
            System.out.println(name);
        }
    }
}
```

- Linked List: Fast insertion and deletion as both ends are open, Slow random access in comparison to Array List
```Java
import java.util.LinkedList;

public class Main {
    public static void main(String args[]) {
        LinkedList<String> history = new LinkedList<>();
        history.add("Google");
        history.add("LeetCode");
        history.add("Youtube");
        history.addFirst("HomePage");
        history.addLast("Github");
        System.out.println(history);
        
        history.removeFirst();
        System.out.println(history);
        history.removeLast();
        System.out.println(history);
    }
}
```

- Vector: thread-safe (rest everything is same as Array list), also called synchronized
```Java
import java.util.Vector;

public class Main {
    public static void main(String args[]) {
        Vector<String> emp = new Vector<>();
        emp.add("Rahul");
        emp.add("John");
        emp.add("Escobar");
        emp.add("Nitin");
        emp.add("Farhan");
        System.out.println("Employees: " + emp);
        
        emp.remove(2);
        System.out.println("Employees: " + emp);

        emp.add("Medhansh");
        System.out.println("Employees: " + emp);

        for(String e : emp){
            System.out.print(e + " ");
        }
    }
}
```
- Stack: LIFO
```Java
import java.util.*;

public class Main {
    public static void main(String args[]) {
        Stack<String> st = new Stack<>();
        st.push("Type A");
        st.push("Type B");
        st.push("Type C");
        st.push("Type D");
        System.out.println("Current Stack: ");
        System.out.println(st);

        System.out.println("Pop");
        st.pop();
        System.out.println("Current Stack: ");
        System.out.println(st);

        System.out.println("Top");
        System.out.println(st.peek());
        System.out.println("Current Stack: ");
        System.out.println(st);
    }
}
```

### Multithreading basics
Threads allow multiple execution to happen at once since each thread is like the smallest unit of execution in Java.  
Two ways to implement threads in Java are: By extending thread class and By implementing Runnable interface.
