## OOP (Object Oriented Programming/Paradigm)
### Is Java a purely OOP language?
No, Java is not a purely OOP language as it doesn't only work on classes and objects, aka only Non-Primitive data types.  
### Wrapper Classes
- Since Java wanted to shift to being a purely OOP language, the primitive data types were converted to wrapper classes under autoboxing.  
(Unboxing is converting wrapper classes into their original formats)  
- This isn't the only reason wrapper classes are used, in DataStructures like ArrayList, the types are not used as "int", rather their wrapper classes, which would be "Integer".
### Wrapper Classes vs Legacy Classes
Classes that were put under Collection in the original version of the JDK (Version 1) are known as legacy classes.
### Two Pointer technique
This technique is used when we have to find either a range, or a sum of that range, find a substring, or find the length of the substring, or unique substring etc.
### Unique sub-string of variable size implementaion
This will be done via HashSet as it doesn't allow duplicates and instead of being of type String, it will be of type Char to allow each character of the string to be stored differently. 
<hr>
The code for this is:   

        HashSet<Character> set = new HashSet<>();
        int left = 0;
        int right = 0;
        int maxLength = 0;

        while (right < str.length()) {
            char ch = str.charAt(right);

            if (!set.contains(ch)) {
                set.add(ch);
                maxLength = Math.max(maxLength, right - left + 1);
                right++;
            } else {
                set.remove(str.charAt(left));
                left++;
            }
        }
