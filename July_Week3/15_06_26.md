## Object Oriented Programming
### 4 Pillars
- Inheritance 
- Polymorphism
- Encapsulation
- Abstraction
### Class, Objects, Functions
- Classes are like blueprints for objects and are imaginary.
- Objects are instances of classes and real world entities
- Functions define the functionality of our objects.
### Polymorphism
Polymorphism is nothing but different methodologies to solve the same question.  
Is of two types  
- Compile Time (Method Overloading): This occurs in the same class, different functions but have the same name but just have different arguments.
- Run Time (Method Overriding): Parent properties are overriden
### Encapsulation
Binding code inside of different entities, such as classes or functions.  
The first step involves making the data members private.  
The next step is to generate their getters and setters.  
### Abstraction
Hiding internal working of any system and showing the user only what we want to show to them.  
There are two ways to achieve abstraction.
- Abstract classes: May or may not have abstract functions in them.
- Interfaces: all functions defined inside of them are abstract by default.
### Collection
Collection contains list, set, queue, which further have:
- Array list
- Linked list
- Vector
- Stack
